# 索引设计规则

> 索引设计决策树、联合索引顺序原则、软删除索引规范、分页索引规范。**Step 3 索引设计前必须加载。**

### 2.1 索引设计决策树

```
查询字段 → 判断是否适合建索引？
    ├── 选择性 < 5%（如性别：男/女）→ 不建单列索引，可考虑与其他字段联合
    ├── 选择性 5%~20% → 视查询频率决定，建议建联合索引
    └── 选择性 > 20% → 适合建索引

索引类型决策：
    ├── 单字段等值查询 → 单列索引
    ├── 多字段等值查询 → 联合索引（选择性高的字段在前）
    ├── 等值 + 范围混合 → 联合索引（等值字段在前，范围字段在后）
    ├── 查询 + 排序 → 覆盖索引（WHERE 字段 + ORDER BY 字段）
    └── 唯一约束 → UNIQUE KEY（兼具约束和索引功能）
```

### 2.2 联合索引字段顺序原则

1. **等值条件字段排在范围条件字段之前**（范围条件会中断最左前缀）
2. **选择性高的字段排在前面**（过滤效果更强）
3. **ORDER BY 字段紧跟在 WHERE 等值条件字段之后**（避免 filesort）
4. **覆盖查询所需的额外字段放在最后**（实现覆盖索引，避免回表）

示例：查询 `WHERE tenant_id = ? AND status = ? ORDER BY created_at DESC`
→ 联合索引：`(tenant_id, status, created_at)` ✅
→ 单独索引：`(tenant_id)` + `(status)` ❌（无法利用索引排序）

### 2.3 软删除索引规范

含 `deleted_at` 字段的表，**不单独对 `deleted_at` 建索引**：
- 高频查询：`WHERE deleted_at IS NULL AND status = ?` → 联合索引 `(status, deleted_at)` 或 `(deleted_at, status)`
- 若数据量 > 1000万且删除率 < 10%，考虑 MySQL 8.0 函数索引：`idx_not_deleted` ON `((deleted_at IS NULL))`

### 2.4 分页索引规范

**浅分页（OFFSET < 10000）**：
- 覆盖索引 + 子查询优化：先通过索引查 id，再回表查数据

**深分页（OFFSET >= 10000）**：
- ⚠️ 必须在设计说明中标注「深分页风险」
- 推荐方案：游标分页（`WHERE id > {last_id} ORDER BY id LIMIT {size}`）
- 若业务必须支持跳页，建议限制最大页码（如不超过第1000页）

### 2.5 禁止建索引的场景

| 禁止场景 | 替代方案 |
|---------|---------|
| 低选择性单列索引（枚举值 ≤ 5个） | 与高选择性字段建联合索引 |
| TEXT / BLOB 字段全列索引 | 前缀索引（`INDEX idx_content (content(100))`） |
| 冗余索引（(A) 和 (A,B) 同时存在） | 删除单列索引 (A)，保留 (A,B) |
| 超过5个字段的联合索引 | 重新审视表设计，可能违反范式 |
| 写多读少的字段（如计数器、余额） | 使用 Redis 缓存，定期同步 |

### 2.6 索引命名规范

```
PRIMARY KEY                 → 无需命名，系统自动
UNIQUE KEY uk_{表名缩写}_{字段}   → 如 uk_user_phone
INDEX idx_{表名缩写}_{字段组合}  → 如 idx_order_tenant_status_created
FULLTEXT ft_{表名缩写}_{字段}   → 如 ft_article_content
```

### 2.7 强制索引场景

> ⛔ 以下场景必须建索引，不得遗漏：

| 场景 | 索引类型 | 说明 |
|-----|---------|------|
| 业务唯一约束（如手机号、订单号） | UNIQUE KEY | 兼具约束和查询加速 |
| 多租户隔离字段（`tenant_id`） | 联合索引首字段 | 所有查询以 tenant_id 为第一条件 |
| 外键关联字段（逻辑外键） | 普通索引 | 关联查询和级联删除性能 |
| 状态字段（频繁按状态筛选） | 与其他字段联合索引 | 单独建低选择性索引无效 |
| 时间范围查询字段（如 `created_at`） | 联合索引末字段或单列索引 | 报表/统计查询 |
| 软删除字段 | 联合索引（非单独） | 见 2.3 规范 |
