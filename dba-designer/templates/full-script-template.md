# 全量脚本结构模板

> 全量初始化脚本与增量变更脚本的结构模板。**Step 5 全量脚本合并前加载。**

### 全量脚本文件结构

```sql
-- ============================================================
-- 项目名称：{项目名}
-- 数据库类型：{MySQL 8.0 / PostgreSQL 14+ / TiDB x.x}
-- 生成工具：dba-designer skill
-- 生成时间：{YYYY-MM-DD HH:mm}
-- 来源文档：
--   - doc/detailed/{模块1}_{功能域1}.md
--   - doc/detailed/{模块2}_{功能域2}.md
-- ⚠️  本脚本为全量初始化脚本，幂等设计，重复执行不产生副作用
-- ⚠️  生产环境执行前请先在测试环境验证
-- ============================================================

-- ============================================================
-- 第0节：建库（按需取消注释）
-- ============================================================
-- CREATE DATABASE IF NOT EXISTS `{项目名}_db`
--   CHARACTER SET utf8mb4
--   COLLATE utf8mb4_unicode_ci;
-- USE `{项目名}_db`;

-- ============================================================
-- 第1节：全局设置（MySQL 专属）
-- ============================================================
SET NAMES utf8mb4;
SET FOREIGN_KEY_CHECKS = 0;
SET SQL_MODE = 'NO_ENGINE_SUBSTITUTION';

-- ============================================================
-- 第2节：建表（按依赖顺序：被依赖的表在前）
-- ============================================================

-- === 基础表（无依赖） ===
-- {表1} DDL 内容...

-- === 业务表（依赖基础表） ===
-- {表2} DDL 内容...

-- === 关联中间表（依赖业务表） ===
-- {中间表} DDL 内容...

-- ============================================================
-- 第3节：初始化数据（种子数据）
-- ============================================================

-- 系统配置初始数据
INSERT IGNORE INTO `{配置表名}` (`key`, `value`, `description`, `created_by`, `updated_by`) VALUES
  ('{key1}', '{value1}', '{说明}', 1, 1),
  ('{key2}', '{value2}', '{说明}', 1, 1);

-- 枚举字典数据（如有）
-- INSERT IGNORE INTO `dict_items` ...

-- 初始管理员账号（如有）

-- ============================================================
-- 第4节：版本记录

-- INSERT IGNORE INTO `schema_version` (`version`, `description`, `executed_at`) VALUES

-- ============================================================
-- 第5节：全局设置恢复（MySQL 专属）
```

### 增量变更脚本模板（增量模式 Step 5.5）

```sql
-- ============================================================
-- 增量变更脚本
-- 项目名称：{项目名}
-- 基础版本：v{N}.{M}.{P}（对应 {项目名}_full.sql 版本）
-- 本次版本：v{N}.{M}.{P+1}
-- 变更内容：{简要描述}
-- 生成时间：{YYYY-MM-DD}
-- ⚠️  执行前请先备份数据库
-- ⚠️  回滚方案：见文件末尾 ROLLBACK 节
-- ============================================================

-- === 执行前检查（确认当前版本正确）===
-- SELECT version FROM schema_version ORDER BY id DESC LIMIT 1;
-- 期望值：v{N}.{M}.{P}

SET FOREIGN_KEY_CHECKS = 0;

-- ============================================================
-- 变更内容
-- ============================================================

-- 新增字段
ALTER TABLE `{表名}`
  ADD COLUMN `{新字段名}` {类型} NOT NULL DEFAULT {默认值} COMMENT '{注释}'
  AFTER `{前置字段名}`;

-- 新增索引
ALTER TABLE `{表名}`
  ADD INDEX `idx_{表名}_{字段}` (`{字段}`);

-- 修改字段（⚠️ 高风险，可能导致数据截断，请先在测试环境验证）
-- ALTER TABLE `{表名}` MODIFY COLUMN `{字段名}` {新类型} NOT NULL DEFAULT {默认值} COMMENT '{新注释}';

-- ============================================================
-- 更新版本记录
-- ============================================================
-- INSERT INTO `schema_version` (`version`, `description`, `executed_at`) VALUES
--   ('v{N}.{M}.{P+1}', '{变更描述}', NOW());

SET FOREIGN_KEY_CHECKS = 1;

-- ============================================================
-- ⬇️ ROLLBACK（回滚方案，不执行，仅供参考）
-- ============================================================
-- ALTER TABLE `{表名}` DROP COLUMN `{新字段名}`;
-- ALTER TABLE `{表名}` DROP INDEX `idx_{表名}_{字段}`;
-- ============================================================
```
