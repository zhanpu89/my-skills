# 术语表（tester 技能）

> 定义 tester 技能中使用的核心术语，供 AI 在生成测试用例、测试代码、测试报告时保持术语一致性。

## 核心术语

| 术语 | 含义 | 使用场景 |
|------|------|---------|
| **阶段一** | 测试用例设计阶段：读取详设文档 → 设计用例 → 输出可评审的 `{模块名}_测试用例.md` | Step 0 阶段识别 |
| **阶段二** | 测试执行阶段：读取已确认用例文档 → 生成测试代码 → 执行测试 → 输出测试报告 | Step 0 阶段识别 |
| **测试用例文档** | 阶段一产出，Markdown 格式，含完整用例 ID / 前置条件 / 步骤 / 预期结果，可被 review-expert 独立评审 | 阶段一输出、阶段二输入 |
| **测试代码** | 阶段二产出，依 LC-001/LC-006 生成的可执行测试骨架，放在 `src/test/` 目录 | 阶段二 Step 2 |
| **测试报告** | 阶段二产出，标准化结论文档，含执行结果 / 缺陷清单 / 通过判定，文件名含 `_RN` 轮次后缀 | 阶段二 Step 4 |
| **LC-001** | 项目规则中的后端语言约束（Java / Python / Go / Node.js），决定测试框架选择 | 全流程 |
| **LC-006** | 项目规则中的测试框架约束（JUnit 5 / pytest / Jest / testing），与 LC-001 配套 | 阶段二 Step 2 |
| **TC-{MOD}-{TYPE}-{SEQ}** | 测试用例 ID 格式：模块缩写 + 类型缩写 + 3位序号，如 `TC-USR-UNIT-001` | 用例 ID 规范 |
| **BUG-{MOD}-{SEQ}** | 缺陷 ID 格式：模块缩写 + 3位序号，如 `BUG-USR-001`，跨轮次不重置 | Step 3.5 缺陷记录 |
| **UNIT** | 单元测试用例类型：测试单个业务规则 / 代码分支，隔离外部依赖 | 用例覆盖策略 |
| **INTG** | 集成测试用例类型：测试接口契约 / 完整业务链路 / 数据一致性 | 用例覆盖策略 |
| **SEC** | 安全测试用例类型：认证 / 越权 / 注入场景，来自第10节和 ER-SECURITY | 用例覆盖策略 |
| **PERF** | 性能测试用例类型：RT / QPS 目标，来自第9节和 ER-PERFORMANCE | 用例覆盖策略 |
| **FE** | 前端测试用例类型：组件渲染 / 表单校验 / API Mock，LC-FE-001 存在时启用 | 用例覆盖策略 |
| **PASS** | 用例执行状态：实际输出与预期结果完全一致 | Step 3 执行结果 |
| **FAIL** | 用例执行状态：实际输出与预期不符，触发 Step 3.5 缺陷记录 | Step 3 执行结果 |
| **BLOCKED** | 用例执行状态：依赖未就绪（环境问题 / 前置接口未完成） | Step 3 执行结果 |
| **SKIP** | 用例执行状态：当前版本暂不执行（需专项环境 / 代码未完成） | Step 3 执行结果 |
| **P0** | 缺陷严重程度：致命——系统崩溃 / 数据丢失 / 安全漏洞，发现后立即暂停测试 | Step 3.5 缺陷分级 |
| **P1** | 缺陷严重程度：高——核心功能不可用 | Step 3.5 缺陷分级 |
| **P2** | 缺陷严重程度：中——功能异常但有绕过方案 | Step 3.5 缺陷分级 |
| **P3** | 缺陷严重程度：低——UI 问题 / 文案错误 / 轻微体验问题 | Step 3.5 缺陷分级 |
| **_PROGRESS.md** | 进度追踪文件，记录各模块用例文档的生成状态（⏳待完成 / ✅已完成），阶段一必须维护 | Step 1 进度管理 |
| **评审专项** | 标注 `[评审专项]` 的用例，来自 code-reviewer 生成的评审报告中的 P0/P1 问题 | Step 1 额外输入 |
| **回归测试** | 多轮测试中，针对上轮 FAIL/BLOCKED 用例的重新执行，报告中新增"上轮缺陷回归情况"节 | Step 4 多轮模式 |

## 文档命名规范

| 产出物 | 命名格式 | 示例 |
|--------|---------|------|
| 测试用例文档 | `{模块名}_测试用例.md` | `用户模块_测试用例.md` |
| 测试报告（首轮） | `{模块名}_测试报告.md` | `用户模块_测试报告.md` |
| 测试报告（第N轮） | `{模块名}_测试报告_RN.md` | `用户模块_测试报告_R2.md` |
| 汇总测试报告 | `测试报告.md` | `测试报告.md` |
| 进度追踪文件 | `_PROGRESS.md` | `_PROGRESS.md` |
| 文档编号（用例）| `TC-YYYYMMDD-NNN` | `TC-20260522-001` |
| 文档编号（报告）| `TST-YYYYMMDD-NNN` | `TST-20260522-001` |

## 测试结论判定标准

| 结论 | 条件 |
|------|------|
| ✅ 通过 | 无 P0/P1 缺陷，P2 ≤ 3 条，PASS 率 ≥ 90% |
| ⚠️ 有条件通过 | 无 P0，有 P1 但 PASS 率 ≥ 80%（需 3 个工作日内修复 P1） |
| ❌ 不通过 | 存在任意 P0，或 PASS 率 < 80% |
