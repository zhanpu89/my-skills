# 记忆加载

会话启动或上下文切换时，恢复历史记忆到当前上下文窗口。

## 触发条件

以下任一条件满足时执行：

1. **新会话开始**：用户打开新对话或说"开始工作"、"继续上次"
2. **项目切换**：用户提到不同项目名称或切换工作目录
3. **用户明确请求**：说"加载记忆"、"恢复上下文"、"上次做到哪了"
4. **匹配进行中任务**：用户描述的任务与某个 `in_progress` 状态的摘要匹配

## 加载流程

### 步骤 1：识别项目上下文

执行"公共前置：项目上下文识别"流程（见 SKILL.md）：
- 从当前目录向上查找 `.project_name` 文件（最多 2 层）
- 读取文件内容获取 `project_name`
- 检查 `.git/HEAD` 获取 `branch_name`（不存在则标记为 `"no-vcs"`）
- 首次读取后缓存，同一会话复用

### 步骤 2：调用 init_session

```
init_session(project_name, branch_name)
```

获取最近 3 天内 `in_progress` 的任务（最多 3 条）。

**响应格式**：
- 成功时返回 `{"success": True, "message": "...", "data": [...]}`
- `data` 字段包含任务列表（无任务时为空数组 `[]`）
- 当有任务时，`message` 描述找到的任务数量

- 存在进行中任务 → 步骤 3
- 没有进行中任务 → 步骤 4

### 步骤 3：恢复进行中任务上下文

对每条 `in_progress` 任务：

1. 调用 `get_summary_by_id(session_id)` 获取完整摘要
   - **响应格式**：`{"success": True, "data": {...摘要对象...}}` 或 `{"success": False, "message": "..."}`
2. 提取关键信息：核心目标、技术决策、问题与方案、涉及文件、下一步计划
3. 向用户展示恢复提示：

```
检测到 {N} 个进行中的任务：
1. {task_title} — 下一步: {next_steps}
2. ...

要继续哪个任务？
```

### 步骤 4：渐进式上下文补充（可选）

根据对话需要触发，不主动执行：

- 调用 `list_recent_sessions(project_name, branch_name, limit=5)` 获取最近完成的任务列表（仅标题+状态，不加载全文）
  - **响应格式**：`{"success": True, "data": [...]}`
- 如果用户提到特定主题，调用 `search_summaries(query=主题, use_vector=True, limit=3)` 进行语义检索
  - **响应格式**：`{"success": True, "data": [...]}`

原则：按需加载，避免一次性占用过多上下文窗口。

## 上下文窗口预算

参考 `glossary.md` 中的上下文层级表（L0–L3）。默认使用 L0，仅在需要时逐级升级。每次升级前评估：加载的信息是否直接有助于当前任务？
